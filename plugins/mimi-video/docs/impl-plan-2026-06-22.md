# mimi-video 优化实施计划(2026-06-22)

> **执行说明:** 本计划在上游 repo `cjhyy/mimi-plugins` 的 worktree `feat/mimi-video-optimize` 内执行,路径根 = `.git-worktrees/mimi-video-optimize/plugins/mimi-video/`。改的是 skill 的 markdown 提示词文件,无测试框架——每个任务的"验证"= 对照验收标准自查 + 用一个迷你例子 dry-run 一遍走查。改好后由用户手动 push 入上游 + 登记 marketplace.json。

**Goal:** 让 mimi-video 产出的提示词按集/按镜头拆成"复制即贴即梦"的自包含投料单,审核能拦住偏薄提示词,导演阐述有结构化兜底——同时保持"不绑定具体模型版本"的设计哲学。

**Architecture:** 三批改动。A=产出组织(seedance-storyboard-skill 集中改,价值最高);B=质量下限(review + director);C=模型选型纯内部一致性纠偏(去竖屏像素/对齐 16:9,不引入未证实外部数字)。每批一组 commit。

**Tech Stack:** 纯 markdown skill 文件(SKILL.md / templates / examples)。无代码、无构建。

**关键约束(全程):**
- **不写死任何具体模型/版本数字。** 目前默认 Seedance 2.0、图片模型不绑定——保持这个哲学。C 批只做"去掉与 16:9 矛盾的竖屏/方形像素 + 对齐画幅硬约束",不引入 NB2/1080p 等未经核实的外部能力。
- **不删 `.cs-meta.json` 出现在 commit 里**——它是安装产物,发布前要排除。
- 单一事实源不漂移:画幅来自 production-bible、生成方式来自 03-generation-plan,逐段卡片是二者的**投影回填**,不让用户手填。

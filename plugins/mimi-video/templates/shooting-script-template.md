# 可拍剧本（Shooting Script）— epNN《作品名》

> 由编剧从小说/剧情/梗概转化而来。这是「拍戏的剧本」：剧本 + 台词 + 场景融合，
> 按集/段落粗切，受**目标时长**硬约束。下游导演分析、分镜都以本文件为准。
>
> **台词规则**：台词写在这里，是全链唯一权威来源。台词最终走**语音**（视频里有声），
> 但画面上**不出现任何字幕/文字**。后期不烧字。
>
> **场景连续性**：同一场景的多个段落尽量相邻，便于后续同场景镜头连续/接续生成；
> 不同场景之间直接硬切。

---

## 时长与容量（最先锁定，用户拍板）

| 项 | 值 | 说明 |
|----|----|------|
| 目标时长 | **N 分钟**（用户开场确定） | 整集成片目标时长，硬约束 |
| 镜头预算 | ≈ 目标秒数 ÷ 平均镜头时长(约 4-5s) | 例：300s ÷ 4.5s ≈ 67 镜；若控制成本只做 40 镜，应接受成片约 3 分钟，不靠拉长单镜凑时长 |
| 剧情节拍 / 核心事件节点 | **约 3-4 个**（5 分钟常用容量） | 开场问题 / 冲突升级 / 转折选择 / 爽点或悬念收束 |
| 叙事段数 | **4-6 段**（完整贯穿剧本） | 起 / 承 / 升级 / 转 / 合，可按剧情需要调整 |

> 内容量必须服从时长。多出来的支线或重场面**砍掉或挪到后续集数**，不是压缩进来。
> 第一版必须是一条清晰主线下的**完整整集剧本**：一个问题 → 连续冲突推进 → 人物选择/反应 → 爽点/悬念收束。
> ⛔ 不要只列分镜点或镜头表；先写观众能顺着看完的整集剧本，再交给导演/分镜拆镜头。

## 主线钩子（一句话）

> 本集要让观众追下去的那个核心问题/悬念是什么。一句话写清。
> 例：「人人都说他是废柴，但他袖中的黑戒到底是什么？」

> 台词用「角色名：」标注；旁白用「旁白：」。所有台词走语音、禁字幕。

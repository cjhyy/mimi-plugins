# 可拍剧本业务审核技能

[技能说明]
    审核编剧产出的可拍剧本 outputs/<ep>/00-shooting-script.md，确保它适合往下做导演分析、
    分镜、生成。此 skill 做业务审核，与 compliance-review-skill（合规审核）配合，两者都
    PASS 才进入导演分析阶段。

[审核对象]
    - outputs/<ep>/00-shooting-script.md

[审核流程]
    第一步：读取可拍剧本 + production-bible.md
    第二步：按验收清单逐项检查
    第三步：输出结果（全过 → PASS；任一不过 → FAIL + 问题清单）

[验收清单]

    时长与容量
        □ 顶部是否锁定了用户确定的目标时长？
        □ 是否算出了镜头预算（目标秒数 ÷ 平均镜头时长）？
        □ 内容量是否匹配预算——没有明显超量塞进太多剧情？
        □ 砍掉的支线是否明确列出（而非偷偷塞回主线）？

    主线钩子
        □ 是否只有一条清晰主线（一个核心问题/悬念）？
        □ 主线钩子是否一句话说得清？
        □ 是否避免了多线并行/每段跳故事？

    起承转合
        □ 四段结构是否齐全、按比例分配？
        □ 「合」是否是爽点或悬念收束（不平淡收尾）？

    台词
        □ 台词是否写在剧本里（唯一来源）？
        □ 台词是否精炼、服务时长（不靠大段台词撑时长）？
        □ 是否标注了「走语音、画面禁字幕」的处理方式？

    场景与可拍性
        □ 每段是否标了场景（地点/时间）+ 在场人物 + 动作？
        □ 同一场景的段落是否尽量相邻（利于后续连续/接续生成）？
        □ 角色名是否全部用 production-bible 花名册规范名？

[输出规范]
    PASS：
        ✅ **剧本业务审核：PASS**
        主线清晰、时长容量匹配、台词到位、场景可拍。

    FAIL：
        ❌ **剧本业务审核：FAIL**
        **问题1**：[具体问题]
        - 位置：[如 起承转合 > 转]
        - 修改方向：[怎么改]
        **问题2**：…

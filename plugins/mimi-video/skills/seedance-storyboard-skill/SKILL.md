# Seedance 2.0 分镜编写技能

[技能说明]
    将导演的讲戏内容转化为 Seedance 2.0 格式的动态视频提示词。每个剧情点 = 一段完整叙事 = 一条提示词 = 一次 Seedance 2.0 生成任务。段落内可包含多组镜头（如蒙太奇、连续切换）。提示词中使用 @引用语法关联人物和场景素材，输出可直接复制到 Seedance 2.0 平台生成视频。

[执行流程]

    第零步（剪辑设计·必做前置）：把剧情点拆成单镜头，定一版可看的成片
        分镜师不是把每个剧情点翻译成一条长提示词，而是先做剪辑设计 + 镜头分解：
        - 读可拍剧本顶部的目标时长 + 镜头预算（短剧默认按约 4-5s/镜估算；如 5 分钟通常约 60-70 镜，若只做 40 镜需接受成片变短）。
        - 选主线：只保留服务主线钩子的内容；与主线无关的不进第一版（即使导演拆了点）。
        - ⚠️ 镜头分解（核心）：把每个剧情点 Pxx 拆成若干**单镜头 Pxx-Sxx**。
          · 一镜一核心动作：每个镜头只承载一个动作/情绪。例：P01 拆成
            P01-S01 推镜碑亮 / P01-S02 手掌特写攥拳 / P01-S03 横扫人群讥笑脸…
          · ⛔ 绝不把一个剧情点的多个动作焊成一条长 prompt（这是"内容跳、糊、对不上"的根源）。
          · 每镜默认 **3-5s**：短剧节奏优先，冲击/反应 3-4s，台词/情绪重点 5s 左右；只有少数强情绪、长对白或关键转场允许 6-8s，并必须说明理由。
          · ⛔ 不要为了凑目标总时长，把普通单镜硬拉到 7-10s；AI 视频单镜过长会拖节奏、漂移、脸变、动作重复。
          · 若目标成片时长较长，优先**增加镜头数量 / 补充反应镜 / 拆细动作链**，不要拉长单镜；镜头总数应由「目标时长 ÷ 约 4-5s」反推，再按制作成本与用户预算取舍。
          · 如果用户给的镜头预算与目标时长冲突，必须在剪辑设计顶部明确提示：选择 A 保持镜头数→成片变短；选择 B 保持目标时长→增加镜头数，不能静默用长镜头硬凑。
        - 排镜头序列：一个清晰问题 → 情绪推进 → 爽点/悬念收束。
        - 场景连续性：同一场景的相邻镜头标"同场景接上一镜"（可走接续生成）；不同场景直切。
        产出：在 02-seedance-prompts.md 顶部写「剪辑设计」+「镜头索引表」
        （主线钩子 + 每个 Pxx-Sxx 的标题/时长/生成方式/用图/接续 + 砍掉的支线）。

    第一步：读取上游产物
        - 先读 outputs/<ep>/00-shooting-script.md（可拍剧本）：台词的唯一来源、主线、时长预算。
        - 先读项目根目录的 production-bible.md（项目单一事实源）：
            · 角色名、@引用对应的人物，一律以花名册里的规范名为准。
            · 首帧图画幅必须等于「渲染规格」里的锁定值（本项目 16:9）；若上游分镜图比例不符，
              先要求重出再编提示词，绝不能拿非 16:9 的图当首帧（详见下方[画幅统一规则]）。
        - 读取导演讲戏本（01-director-analysis.md）
        - 读取人物提示词（assets/character-prompts.md）
        - 读取场景道具提示词（assets/scene-prompts.md）
        - 读取 seedance-prompt-methodology.md
        - 读取 examples/seedance-prompt-examples.md，了解本项目推荐的提示词风格和结构

    第二步：建立素材对应表
        - 根据人物和场景提示词文件，为本集涉及的素材分配 @引用编号
        - 只有在 character-prompts.md 和 scene-prompts.md 中有提示词的人物/场景才分配 @引用
        - 不在素材文件中的人物（群演、一次性配角）不进对应表，在提示词中直接用文字描述
        - 在文档头部建立对应表
        - 具体格式见 templates/seedance-prompts-template.md

        场景素材的编号规则（重要）：
        - scene-prompts.md 中的九宫格是"生成格式"——用于一次性生成场景参考图
        - 生成后，用户会将九宫格中的每个格子单独提取出来，作为独立的场景参考图
        - 因此在素材对应表中，每个场景必须独立编号为一个 @图片，不能将整张九宫格作为一个 @图片
        - 编号顺序：先人物（按 character-prompts.md 顺序），再场景（按 scene-prompts.md 中的格子顺序）
        - 示例：如果有 4 个人物 + 9 个场景，则 @图片1-4 为人物，@图片5-13 为场景

        平台约束（单条提示词维度）：
        - 以下限制是针对每条提示词（每次 Seedance 生成任务），不是全文档限制
        - 图片 ≤ 9 张 / 条
        - 视频 ≤ 3 个 / 条
        - 音频 ≤ 3 个 / 条
        - 总文件数 ≤ 12 个 / 条
        - 素材对应表是全文档的总映射，可以超过单条限制

    第三步：为每个**单镜头 Pxx-Sxx** 编写一条 Seedance 2.0 提示词
        对镜头索引里的每个镜头，各写一条独立短提示词（60-100 词，一镜一核心动作）：
        - 运镜：该镜头的一个主运镜（只一个，且与主体动作分开写）→ Seedance 运镜语言
        - 动作：该镜头的一个核心物理动作 → 具体描述（不堆多个动作）
        - 台词/声音：从 00-shooting-script 取该镜对白 → 语音/旁白/环境音（画面禁字幕）
        - 光影：光源方向/色温/明暗（灯光是最高杠杆，必写）
        - 约束：每条结尾加标准负向约束（无字幕、人物一致、无抖动、16:9）
        每个镜头单独写进 prompts/Pxx-Sxx.md（一镜一文件），不要合并。
        ⛔ 不要把一个剧情点的多镜头合成一条——那是旧版"内容奇怪"的根因。

    第四步：输出提示词（双层产出）
        按 templates/seedance-prompts-template.md 的双层格式输出：
        - 层1 总览索引 outputs/<集数>/02-seedance-prompts.md：素材对应表(含本地文件名列) + 分段索引。
        - 层2 每段独立投料单 outputs/<集数>/prompts/Pxx.md：投料单卡片(生成方式/画幅/本段用图) + ```text 代码块包裹的提示词正文。
        每段 Pxx.md 必须自包含——用户复制该文件的提示词块即可直接贴进即梦,无需翻其他文件。
        卡片里的生成方式/画幅/用图,在第五步生成计划确认后回填,不让用户手填。

    第五步：与用户确认「生成计划」（提示词通过审核后进行）
        提示词只是"要生成什么"，生成计划是"怎么生成"。这一步把生成方式定下来，
        产出一份可执行的生成计划表，交给用户去调工具实际出图/出视频。

        本 skill 不绑定、不预设任何具体出图/视频模型——模型由用户在运行时选择。
        本步骤的职责是「按规则给出推荐 + 引导用户拍板」，需要逐项确认以下决策：

        (1) 出图模型 / 视频模型
            - 询问用户本批用什么模型出分镜图、什么模型生成视频。
            - 不替用户预设默认模型；如用户已说明则记录，未说明则明确问。

        (2) 每个剧情点配几张分镜图（按规则推荐，用户决）
            推荐规则（基于该段的镜头数 / 时长 / 生成方式）：
            - 单镜头段落 → 推荐 1 张首帧图。
            - 首尾帧生成 → 推荐 2 张（首帧 + 尾帧）。
            - 多镜头 / 蒙太奇段落 → 按关键镜头数推荐（每个关键镜头 1 张首帧），
              但受平台单次生成图片数上限约束（图片 ≤ 9 / 条）。
            - 续接上一段视频的段落 → 可不出新首帧图，用上一段视频尾帧续接。
            给出推荐张数 + 理由，最终由用户确认或调整。

        (3) 每段视频的生成方式
            从以下方式中为每段推荐其一（依据该段在分镜中的叙事衔接关系）：
            - 首帧图生视频：用一张首帧图驱动（最常用）。
            - 首尾帧：首帧 + 尾帧，模型补全中间（适合明确的状态转变）。
            - 续接 / 延长：承接上一段已生成视频向后（或向前）延长（适合连续动作）。
            - 参考运镜 / 参考视频：以首帧定画面 + 参考另一段视频的运镜或特效。
            给出推荐方式 + 理由，最终由用户确认。

        (4) 画幅 / 分辨率复核
            - 全部分镜图必须等于 production-bible 渲染规格里的画幅（本项目 16:9）。
            - 视频分辨率按渲染规格（图生视频继承首帧画幅，分辨率单独设）。
            - 若发现任何一段的首帧图比例不符，先回到出图阶段重出，不得带病进入生成。

        输出：按 templates/generation-plan-template.md 产出一张「生成计划表」，
        逐剧情点列出：生成方式 / 分镜图张数 / 用到的 @引用 / 出图模型 / 视频模型 / 画幅。

    第六步：生成执行（逐段确认，严禁批量齐发）⚠️ 硬约束

        ⛔ 绝不能在一轮里把整集所有段落的 GenerateVideo 一次性发出去。
           视频生成是花钱、耗时、难撤销的操作；批量齐发会让用户来不及取消。
           （这正是本 skill 要防的事故：一轮并发十几条视频任务。）

        生成视频的默认归属是用户：交付投料单 + 生成计划表后，用户可以自己拿提示词去出视频。
        仅当用户明确要求「帮我生成视频 / 开始出视频」时，才由本流程代为调用 GenerateVideo，
        且必须严格遵守逐段串行：

        每一段，按这个循环走，一段做完才能开始下一段：
        1. 先用 AskUserQuestion 向用户确认这一段：展示段号 + 提示词摘要 + 用图 + 生成方式，
           问「这一段现在生成吗？」。用户确认后才继续。
        2. 确认后，本轮只调用 1 个 GenerateVideo（这一段）。一轮最多 1 个，绝不并发多个。
        3. 等这一段的后台任务完成通知（带回视频 URL）。
        4. 若该段需要承接上一段（连续动作/运镜延续），用 videos:["上一段返回的 http(s) URL"]
           接续生成（Seedance 模型才支持 reference-to-video；URL 可能过期，拿到后尽快用）。
        5. 回到第 1 步处理下一段。

        硬性禁止：
        - ⛔ 同一轮发出 ≥2 个 GenerateVideo。
        - ⛔ 未经用户逐段确认就开始生成。
        - ⛔ 把所有段落排成一批任务丢给后台。

[Seedance 2.0 提示词写法核心规则]

    叙事描述式
        - 用完整的导演式段落描述，不要关键词堆叠
        - Seedance 2.0 擅长理解长段叙事性提示词
        - 参考 examples/seedance-prompt-examples.md 的本项目原创范例：连续段落，融合运镜、动作、光影、声音

    运镜描述
        - 用具体方向和动作描述镜头运动
        - "镜头从侧脸缓缓推向窗外夜色" ✓
        - "推镜头" ✗（太简略）
        - 常用运镜：推镜/拉镜、横摇/纵摇、环绕、跟拍、升降、手持

    动作描述
        - 用具体的物理动作，避免抽象概念
        - "她揉了揉太阳穴，目光从屏幕移开，靠向椅背" ✓
        - "她感到疲惫" ✗

    时长控制
        - 优先遵循短剧剪辑节奏：普通单镜默认 3-5s，不把镜头拉长来凑总时长。
        - 导演建议时长只作为段落容量参考；若导演时长会导致普通单镜超过 6s，应通过增加镜头数、拆分动作、补反应镜解决，或在剪辑设计中提示用户二选一。
        - 底层规律：每个连续镜头内 1 拍 ≈ 2.5 秒屏幕时间，节拍太密动作会糊、跳、不自然
        - 多镜头段落：节拍分布在多个镜头切换中，每个镜头各自遵守密度约束，总节拍可超单镜头上限
        - 头尾安全区：每次 Seedance 生成前 0.5s 和后 0.5s 为安全区，不放关键内容

    声音/台词（重要规则）
        - 台词来自可拍剧本 00-shooting-script.md（唯一来源），原样取用，不自行改写。
        - 台词**写进 video prompt**，让模型生成对应的语音/对白（有声）。
        - 但必须在 prompt 里**明确要求画面不出现任何字幕/文字**（有声无字）。
          例：在 prompt 末尾加「台词只作语音，画面不显示任何字幕/文字」。
        - 台词用引号标注；可指定语气/声音特征。
        - 可指定背景音乐风格、环境音。
        - 台词精炼、服务时长，不靠大段台词撑时长。

[群演/一次性配角处理]
    导演讲戏本中可能提到不在 character-prompts.md 中的人物（群演、一次性配角）。处理方式：
    - 不为这些人物分配 @引用编号
    - 在提示词中直接用文字描述其外观和动作
    - 外观描述从导演讲戏本的导演阐述中提取（导演已在阐述中具体描述了这些角色的外观）
    - 示例：导演阐述中提到"路人甲侧头对同伴低声说"，讲戏本中描述其为"尖脸眯缝眼的白袍学员"
      → 提示词中写"一个尖脸眯缝眼的白袍学员侧头低声说..."，不使用 @引用

[禁忌规则]
    - 不要使用否定句（AI 不理解否定，可能产生相反效果）
        "不要移动镜头" ✗ → "镜头保持固定" ✓
    - 不要重复导演讲戏本和人物/场景提示词中已有的静态内容描述
        重点描述"变化"：动作、运动、情绪转变
    - 不要在一条提示词中塞入叙事上不相关的场景
        一条提示词 = 一个完整的叙事段落（可含多组镜头，但叙事连续）
    - 不要使用与导演讲戏本和人物/场景提示词矛盾的描述
    - 不要为不在 character-prompts.md 中的人物使用 @引用

[画幅统一规则（硬约束，最重要）]
    - Seedance 图生视频以"首帧图"的宽高比为准生成视频画幅。首帧图比例不统一，
      成片就会横竖混排，无法直接拼接。这是本项目最容易踩的坑，必须从源头锁死。
    - 本项目所有片段统一为 16:9 横屏。每段提示词所引用的首帧图（@图片N）必须是
      16:9（1536×1024）。若上游分镜图里出现非 16:9 的图，先要求重出，再编提示词，
      绝不能直接拿竖图当首帧。
    - 每段投料单卡片的「画幅·分辨率」是硬性字段,必须填(从 production-bible 渲染规格内联,本项目 16:9·1536×1024),不留空、不依赖默认。
    - 整集定位为竖屏短视频时，反过来全集统一 9:16，同样禁止同集内横竖混用。

[注意事项]
    - seedance-prompt-methodology.md 是核心方法论，必须遵守
    - 参考 examples/seedance-prompt-examples.md 了解本项目推荐的提示词风格和结构

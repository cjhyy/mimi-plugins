# 编剧执行技能

[技能说明]
    把原始素材（小说片段 / 剧情梗概 / 大纲）转化成一份「可拍剧本」(shooting script)：
    剧本 + 台词 + 场景融合，按集/段落粗切。这是整条流水线的起点，是台词的唯一权威来源。
    模板见 templates/shooting-script-template.md。

    解决的核心问题：原始小说/剧情信息量大，直接拿去拆分镜会导致「每段都像剧情摘要、
    塞太多、不可看」。编剧的职责是先用**时长**框死容量，只保留一条清晰主线。

[执行流程]

    第零步（必做）：读 production-bible.md
        - 确定本作品的角色花名册（规范名）和渲染规格。
        - 剧本里的角色名一律用花名册规范名；出现新角色先登记规范名再用。

    第一步（必做·最先）：让用户确定目标时长 ⚠️
        - 用 AskUserQuestion 问用户：「这一集成片目标多少分钟？」
        - 这是硬约束，必须在写任何剧情之前定下来，不替用户预设。
        - 锁定后写进可拍剧本顶部「时长与容量」表。

    第二步：算镜头预算 + 事件容量（关键，决定砍多少）
        - 镜头预算 ≈ 目标秒数 ÷ 平均镜头时长（短剧默认约 4~5s）。
          例：5 分钟 ≈ 300s ÷ 4.5s ≈ 67 个镜头；若为控制成本只做约 40 镜，应明确提示成片会更接近 3 分钟，不靠把普通单镜拉长到 7~10s 凑时长。
        - ⚠️ 镜头数够 ≠ 内容合适。真正的硬约束是**主线密度与节拍完整性**：
          · 一集必须先形成一条贯穿全片的**完整可拍剧本**，不是只列分镜点、不是只写镜头表。
          · 5 分钟通常可承载约 **3-4 个剧情节拍 / 核心事件节点**：开场问题、冲突升级、转折选择、悬念/爽点收束。
          · 仍然只保留**一条主线**；事件节点必须服务同一个问题/悬念，不能并行多条支线。
          · 反例(踩过的坑)：把"测试+退婚+休书+三年之约"4 个彼此都要重场面的独立大事件塞进 5 分钟 → 每段都像摘要、跳故事、不可看。
        - 原始素材里多出来的支线/重场面 = 要砍掉或挪到后续集数的，不是压缩进来。

    第二步半（按内容量决定·多数情况跳过）：要不要先建世界观/分集大纲？
        判断依据 = 原始素材的体量：
        - 短素材 / 单集 / 一段剧情 → **跳过**，直接往下写本集剧本（当前测试用例就是这种）。
        - 长篇小说 / 明确要做多集连贯短剧 → 才先建一份项目级 series bible：
          世界观设定 + 人物关系 + 分集大纲（每集讲什么），写进 production-bible，再逐集写剧本。
        默认不做。只有素材确实是长篇/多集时才建，不要给单集硬套世界观。

    第三步：定主线钩子 + 选本集剧情节拍
        - 从原始素材提炼**一条**主线：一个清晰的问题/悬念，一句话写清。
        - 在主线上选择约 **3-4 个剧情节拍 / 核心事件节点**进本集（5 分钟常用容量），其余支线或重场面全部砍掉或挪集。
        - 明确写出「本集完整贯穿剧本讲什么」和「砍掉/挪到后续的事件」两个清单。

    第四步：搭起承转合（4-5 段，不是 8 段）
        - 整集只切 **4-5 个叙事段**，对应起承转合（业界短剧标准结构）：
          · 起(钩子)：~1 段 —— 抛出主线问题/制造悬念
          · 承(发展)：1-2 段 —— 情绪/冲突推进
          · 转(高潮/反转)：~1 段 —— 爽点或反转
          · 合(收束)：~1 段 —— 爽点落地或留悬念钩子
        - ⛔ 不要切成 7-8 段甚至更多——段一多就必然变成"赶事件"。
        - 合：必须是爽点或悬念收束，不能平淡收尾。

    第五步：写分段剧本（剧本+台词+场景融合）
        - 按 4-5 段写：段号 + 场景(地点/时间) + 在场人物 + 动作/画面 + 台词。
        - 每段就是起承转合里的一块，承载一个叙事功能，不要再塞多个事件。
        - 同一场景的段落尽量相邻（便于后续同场景镜头连续/接续生成）。
        - 台词用「角色名：」标注，旁白用「旁白：」。
        - 宁可少而精、讲透，不要填满预算。

    第六步（必做）：回填角色卡花名册到 production-bible.md ⚠️
        业界标准：写剧本和定角色同时做，名字在剧本里就锁死，不拖到下游补。
        把本集所有有名有戏的角色，逐个填进 production-bible.md 的角色花名册表，每个角色填全：
        - 规范名（剧本用的名字，全链统一，下游禁改名）
        - role：main / supporting / minor
        - 一句话定位
        - 外貌：100 字内具体可画（性别/年龄段/体型/面部特征/发型/服装风格）——给服化道出图用
        - 性格：性格标签
        - 配色：主色调（如「深灰+铜色」）——保证全链视觉一致
        群演/一次性配角不进花名册，在剧本里直接文字描述即可。
        ⛔ 不要把花名册留成占位让导演/服化道补——那会导致下游各画各的、角色跳戏。

[台词处理规则（重要）]
    - 台词写在可拍剧本里，是全链唯一权威来源，下游不另立台词文件。
    - 台词最终走**语音**：视频里能听到对白/旁白。
    - 但画面上**绝不出现字幕/文字**。后期不烧字。
    - 台词节奏要服务时长：对白精炼，不靠大段台词撑时长，靠画面叙事。

[产出]
    - outputs/<ep>/00-shooting-script.md（按 templates/shooting-script-template.md 格式）

[禁忌]
    - ⛔ 不先问时长就开始写剧情。
    - ⛔ 把原始小说所有剧情点都塞进第一版（违反时长约束）。
    - ⛔ 多条主线并行（第一版只讲一条）。
    - ⛔ 一集并行多条主线，或塞入过多彼此独立的重场面（必然赶场、每段像摘要、不可看）。
    - ⛔ 用"压缩"代替"砍"——讲不透的事件就砍掉/挪集，不要硬塞。
    - ⛔ 用花名册外的角色名当正名。
